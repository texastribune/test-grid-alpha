# The Grid ## About This grid is a hybrid of Bootstrap, Foundation, and probably some other things. I really like how terse the bootstrap syntax is. But I really hated how it relied on inefficient css selectors. In order to accomodate grid layouts with borders, the gutter is evenly divided. The columns are always fluid, using percents instead of derived pixel widths. This means nesting is automatic, and the layout is more tolerant to browser zoom. ## How to use this demo This code should be considered alpha. Things are broken, missing, undocmented. You're expected to go directly to the source to figure out what's going on. Your web inspector is your friend. There is no reset.css or normalize.css here. One handy thing that is built in is when you hover over content, the background will highlight. If you scan your mouse over many blocks, you can get the highlight to persist. Use this to figure out the exact area that's available to you inside the grid. For most of the blocks below, gray areas have been filled in to help you visualize how the grid is filled. ## TODO Responsive logic. Some grid frameworks don't have this problem because their gutters aren't symmetrical, which makes removing their gutter easier. Other grid frameworks just ignore this. If we want to have the ability to draw lines, we need symmetic gutters.
